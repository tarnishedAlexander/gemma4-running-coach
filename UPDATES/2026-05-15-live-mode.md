## TL;DR

The `litert-lm/` iOS app (GemmaCoach) now supports **fully on-device live coaching**: hit a single toggle and the app fires Gemma 4 every N seconds (5–60s slider), streams the response straight into the iPhone's TTS, and keeps running with the screen off. Mic capture for voice prompts also wired. Both `litert-lm/` and `llama-cpp/` paths still build clean for `generic/iOS`.

If you only read one section, make it [Try it](#try-it).

## What changed since the last update

### Live coaching mode — `litert-lm/ios-app/GemmaCoach/`

Three new Swift files plus extensions to existing ones. Roughly 275 lines added.

- **`CoachSpeaker.swift`** — wraps `AVSpeechSynthesizer`. As the LLM streams tokens, accumulated text gets flushed to TTS at sentence boundaries (`.!?\n`) or after 800ms of silence. Athlete hears the coach talking *as the model generates*, not after. Voice: Samantha, rate 0.52 (slightly slower than default for clarity at running pace).
- **`LiveSession.swift`** — the trigger loop. Toggle on → fires once immediately, then every N seconds. Each new trigger cancels any in-flight inference and any in-flight TTS so you never get "ghost coach" overlap. Manages `AVAudioSession(.playback, .spokenAudio, .duckOthers)` so background music ducks during coaching and resumes after.
- **`EngineModel.streamCoach(prompt:onChunk:)`** — clean streaming bridge with proper `Task.isCancelled` checks per chunk.
- **`UIBackgroundModes: [audio]`** added in `project.yml` → app survives screen-off and foreground swaps to Maps mid-run.
- **`ContentView` `liveCard`** — Live toggle, period slider (5–60s), live metrics (trigger count, last decode tok/s, time since last fire). Speaker icon shows when TTS is actively talking.

### Audio input — `litert-lm/ios-app/GemmaCoach/`

Wired in the previous push (`c79b666`):

- **`AudioRecorder.swift`** — `AVAudioRecorder` writing 16 kHz mono WAV (matches what LiteRT-LM resamples to internally).
- **Mic permission** prompt fires on first record.
- **UI:** record/stop button with live audio meter (smoothed dB → 0..1 progress bar), "audio ready" indicator after stop.
- **Engine methods:** `generateAudio(audioData:prompt:)` and `generateMultimodal(audioData:imagesData:prompt:)`.
- **Action row:** four buttons — *Text* / *+image* / *+audio* / *All* (full multimodal).

### Testing parity doc — `litert-lm/TESTING_PARITY.md`

Three real ways to get closer-to-iPhone numbers than the iOS Simulator (which overestimates by 3–4× on M4):

- **Path A** — Xcode Devices Over Network: pair any iPhone you own over WiFi → real device, real perf, free.
- **Path B** — BrowserStack App Live: cloud iPhone 16 Pro in your browser, $49/mo (or 30-min free trial). Upload our unsigned `.app`.
- **Path C** — macOS-side throttling: thread caps + memory limits for relative perf comparison during dev iteration.

Plus a reference table of published Gemma 4 E2B numbers across 5 backends (llama.cpp Metal, LiteRT-LM GPU, LiteRT-LM with MTP, Google AI Edge Gallery, CoreML-LLM ANE) so any number we quote in a deck is defensible.

## Try it

### Build

On the Mac mini:
```
cd ~/work/evan/gemma4-running-coach/litert-lm
./apply.sh
```

That installs XcodeGen if needed, regenerates `GemmaCoach.xcodeproj`, resolves the `LiteRTLM-Swift` Swift Package dep, and builds for `generic/iOS`. Output: `ios-app/build_ios/Build/Products/Debug-iphoneos/GemmaCoach.app` (40 MB — model isn't bundled, app downloads on first launch).

### Deploy

```
open ~/work/evan/gemma4-running-coach/litert-lm/ios-app/GemmaCoach.xcodeproj
```
- Signing & Capabilities → Automatically manage signing → pick Team
- Plug in iPhone 13 Pro or newer (8 GB RAM ideal) → select destination → ⌘R
- First launch downloads `gemma-4-E2B-it.litertlm` (~2.6 GB) from HF
- Once "Ready" — try:
  - **Text** button: types text-only prompt
  - **📷 photo button** then **+image** for vision
  - **🎤 record button** then **+audio** for voice prompts
  - **All** for full multimodal
  - **Live mode toggle** (bottom card): set period 10s, flip on, lock screen, listen

### What live mode looks like in use

1. Set period slider to e.g. 15s
2. Flip Live toggle on
3. The app immediately fires the first inference; coach starts speaking via Samantha voice within ~2-3 sec
4. Lock the iPhone or switch to Maps — coach keeps talking on schedule
5. Every 15s, a new coaching line plays; previous one cuts off if still mid-sentence
6. Flip toggle off — silence, audio session releases, music resumes

## Known caveats / what's NOT done

- **Not yet tested on a real iPhone.** Build succeeds for `generic/iOS` and the embedded LiteRTLM symbols verify, but no one has run it on hardware. First real test should be Path A from `TESTING_PARITY.md` (5-min setup if you own any iPhone).
- **No HealthKit yet.** The live-mode prompt template is a static "be a coach" string. Real value comes when the prompt is filled per-trigger with current pace / HR / cadence — that needs ~30 min of HealthKit wiring, not yet done.
- **No Apple Watch companion.** Discussed; not built.
- **CoreML-LLM/ANE path** (`coreml-ane/`) is still on the table for the production version (lower power, lower RAM, 2× decode tok/s on iPhone via ANE), but explicitly skipped for now in favor of getting the GPU path tested.
- **Live mode period defaults to 15s.** Hard-coded text prompt, no incorporation of camera/audio per trigger yet — those are easy adds (LiveSession can grab the latest frame/audio buffer per fire).
